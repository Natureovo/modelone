# 视频测试数据集说明

这个目录用于后续对视频进行批量测试，尤其适配使用 HM 对视频进行压缩后的结果整理。

## 推荐流程

1. 将未压缩或高质量源视频放入 `raw_videos/`。
2. 使用 HM 对视频进行不同 QP 压缩，保留码流、重建 YUV 和日志。
3. 将原始视频与 HM 重建结果抽成一一对应的视频帧。
4. 使用本项目对 `原始帧` 与 `压缩帧` 做梯度域和频率域细节损失分析。

## 目录含义

```text
dataset/
  raw_videos/              原始视频或原始 YUV
  hm_results/
    qp_22/                 HM QP22 结果
    qp_27/                 HM QP27 结果
    qp_32/                 HM QP32 结果
    qp_37/                 HM QP37 结果
    qp_42/                 HM QP42 结果
    bitstreams/            HM 输出码流，例如 .bin
    recon_yuv/             HM 重建视频，例如 .yuv
    logs/                  HM 编码日志
  frame_pairs/
    视频ID/
      原始帧/              从原始视频抽出的帧
      HM_QP32_压缩帧/      从 HM 重建视频抽出的对应帧
  splits/                  训练/验证/测试划分
  metadata.csv             视频级元数据
  frame_pairs.csv          帧级配对清单
```

## 帧命名建议

原始帧和压缩帧必须同名，便于批量匹配：

```text
000001.png
000016.png
000031.png
```

例如：

```text
frame_pairs/视频001/原始帧/000001.png
frame_pairs/视频001/HM_QP32_压缩帧/000001.png
```

## 建议 QP

```text
QP22：轻度压缩
QP27：中轻度压缩
QP32：中度压缩
QP37：重度压缩
QP42：极重压缩
```

## 与本项目输出的关系

本项目现在会输出 `区域损失明细.csv`，里面每一行对应一个图像块区域，包含：

- 区域坐标 `x, y, width, height`
- 梯度损失分数
- DCT 高频损失分数
- DCT 高频能量损失比例
- 综合细节损失分数
- 区域建议：高频生成候选、纹理生成候选、结构保真修复、优先去块效应等

这比只看整图平均值更接近“判断哪些区域高频损失严重”的研究思路。
